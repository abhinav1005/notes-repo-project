# Knowledge Hub

A personal knowledge management system. Write Markdown notes locally, push to GitHub, and a CI pipeline automatically tags and summarizes them with Claude AI, then deploys the updated site to Vercel.

---

## How It Works

```
Write .md file → push to main → GitHub Actions triggers
    → Claude generates tags + summary
    → entries.json committed back [skip ci]
    → Vercel detects commit → rebuilds → live site updated
```

Everything is static at runtime. No database, no runtime API calls — the pipeline pre-processes notes into `entries.json` which Vite bundles into the frontend.

## Writing Notes

Create a `.md` file in the `notes/` folder:

```markdown
---
title: "My Note Title"
type: note
sourceUrl: "https://example.com"   # optional
---

## Section Heading

Your content here. Full GitHub Flavored Markdown supported.

### Subsection

- bullet lists
- **bold**, _italic_, `inline code`

| Column A | Column B |
|---|---|
| value    | value    |

​```mermaid
graph TD
  A --> B --> C
​```
```

**Frontmatter fields:**

| Field | Required | Notes |
|---|---|---|
| `title` | Yes | Display title |
| `type` | No | `note` \| `paper` \| `blog` \| `discussion` \| `resource` — defaults to `note` |
| `sourceUrl` | No | Link shown on the detail page |

`tags` and `summary` are **auto-generated by Claude** — don't add them manually.

**Diagrams:** Use standard Mermaid fenced code blocks (` ```mermaid `). Sequence diagrams, flowcharts, and graph diagrams all work.

## Entry Schema

The pipeline produces `entries.json` with this shape:

```typescript
type Entry = {
  entryId:    string    // slugified filename, e.g. "url-shortener"
  type:       "note" | "paper" | "blog" | "discussion" | "resource"
  title:      string
  tags:       string[]  // auto-generated (3–6 kebab-case tags)
  summary:    string    // auto-generated (≤50 words)
  content:    string    // raw markdown body
  sourceUrl?: string
  sourceFile: string    // "notes/url-shortener.md"
  createdAt:  string    // ISO 8601 — set once, never overwritten
  updatedAt:  string    // ISO 8601 — updated on every pipeline run
}
```

## Pages

| Route | Description |
|---|---|
| `/` | Grid/list view with type filters and load-more pagination |
| `/entry/:id` | Full note with rendered Markdown, Mermaid diagrams, and sticky ToC |
| `/tags` | Tag cloud — click to filter entries by tag |
| `/search` | Real-time client-side search across title, summary, tags, and content |

| Key | Action |
|---|---|
| `⌘K` / `Ctrl+K` | Open search modal |
| `/` | Open search modal |
| `↑` `↓` | Move up/down (grid rows, list items) |

## Local Setup

**Frontend**

```bash
cd knowledge-hub
npm install
npm run dev        # http://localhost:5173
npm run build      # production build → dist/
```

**Pipeline (run locally)**

```bash
# create .env in repo root
echo "ANTHROPIC_API_KEY=sk-ant-..." > .env

pip install -r backend/requirements.txt

# process all notes
python backend/process_notes.py

# or process specific files
CHANGED_FILES="notes/my-note.md" python backend/process_notes.py
```

**Backend API (optional, for future RAG)**

```bash
cd backend
uvicorn main:app --reload   # http://localhost:8000
# GET  /health
# POST /api/ask  (stub — not yet implemented)
```

## Deployment

### GitHub Actions

The workflow (`.github/workflows/process-notes.yml`) runs automatically when any `notes/*.md` file is pushed to `main`.

Required secret → **Settings → Secrets → Actions**:

```
ANTHROPIC_API_KEY   sk-ant-...
```

### Vercel

1. Connect the GitHub repo to Vercel
2. Set root directory to `knowledge-hub/`
3. No additional environment variables needed (the pipeline runs in Actions, not Vercel)

`vercel.json` already configures the SPA rewrite so React Router routes work in production.
